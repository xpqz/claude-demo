## Design decisions

1. Title extraction handles both form-urlencoded (nested vector, depth > 1, needs disclose) and JSON (simple string, depth 1, use as-is) payloads. Checked via `1<≡title`.

2. Missing-title check uses a trapped `⎕NC` call: `{0::1 ⋄ 0=⍵.⎕NC⊂'title'}req.Payload`. When the request body is empty, `Payload` may not be a namespace (Jarvis leaves it as empty string), and `⎕NC` would error. The trap catches this.

3. Status 201 is set via `req.Response.Status←201`, not `req.StatusCode←201`. Jarvis reads `Response.Status` when building the HTTP response; `StatusCode` is a field on the request object that nothing reads.

4. Integration tests use `⎕NEW HttpCommand` with explicit `ContentType←'application/x-www-form-urlencoded'` and namespace params. The `HttpCommand.Do` shared method with simple string params has encoding issues (spaces cause 400s from Jarvis). Using instance-based calls with namespace params lets HttpCommand handle URL-encoding correctly.

5. Test port is randomised (`18900+?100`) to avoid port conflicts from lingering Conga connections between test runs.
