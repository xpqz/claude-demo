# Issue #4: Phase 3 -- Static files

Branch: `4-static-files`

## Summary

Front-end assets for the todo application: the HTML shell page and CSS stylesheet. Both are plain files in `static/`, served by APL handlers in later phases via `⎕NGET`.

## Changes

### New files

| File | Description |
|------|-------------|
| `static/index.html` | Shell page. Loads htmx 2.0.4 from CDN, links `/style.css`. Contains a form that POSTs to `/todos` with htmx attributes for in-place append and reset. Contains `<ul id="todo-list">` that loads todos on page load via `hx-get="/todos"`. |
| `static/style.css` | Minimal layout (centred, max-width 600px). Completed todos get strikethrough via `.completed span`. Delete buttons styled as borderless red text. `.error` class for error fragments. |
