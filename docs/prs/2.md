## Design decisions

1. **MockRequest is purpose-built, not a full Jarvis clone.** Stark's MockRequest has a `Body` field; ours omits it because the todo application never uses it. We add `SetContentType` and `ContentTypeForFile` which Stark's mock lacks.

2. **`Fail` follows Stark's guard pattern.** `Fail 0` is a no-op (does not set `Response.Status`), matching Stark's behaviour at `MockRequest.aplc:26`. This preserves compatibility if downstream code relies on it.

3. **`ContentTypeForFile` uses `ŌÄĢNPARTS` for extension extraction.** The third element of `1 ŌÄĢNPARTS path` gives the file extension including the leading dot. We drop the dot and lowercase via `ŌÄĢC`. Files with no extension (empty third element) fall through to `application/octet-stream`.

4. **`SetContentType` stores headers as enclosed strings.** `Response.HeadersŌŹ¬ŌåÉ` creates enclosed elements. Tests use `ŌŖā` (disclose) when comparing individual cells, matching how Jarvis matrices behave.

5. **`RunHelper.apln` is imported into `#` directly** (not `#.RunHelper`) to match Stark's import pattern. `Link.Import` creates the namespace from the file name.
