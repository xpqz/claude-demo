### New methods on TodoApp

- FindTodo: parses id via VFI (rejects multi-token, non-numeric), checks empty todos vector, searches by id, returns index or -1 with appropriate Fail status.
- ToggleTodo: calls FindTodo, flips completed, returns updated TodoItem fragment.
- DeleteTodo: calls FindTodo, removes by index, returns empty string via RespondHTML.

## Design decisions

1. FindTodo handles empty todos vector explicitly (returns 404) to avoid SYNTAX ERROR from `todos.id` on an empty vector.

2. Integration tests extract the todo ID from the POST response HTML (parsing `<li id="todo-N"`) rather than assuming `id=1`. This makes tests independent of execution order, since shared server state means earlier tests may have already created items.

3. Deletion uses index-based removal: `((⍳≢todos)≠idx)⌿todos`. This is clearer than the reference-identity approach (`todos~⊂idx⊃todos`).
