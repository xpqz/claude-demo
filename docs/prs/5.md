## Design decisions

1. TodoApp uses `#.Stark` instead of `##.Stark` for the Stark reference. Link.Import may place the class in a different namespace tree depending on import order, making `##` resolution unreliable. `#.Stark` is explicit and safe.

2. The Stark OnErrorFn validation at Start time uses `Handlers.NC` to check the error handler exists and `Handlers.AT` to verify its signature. Both fail for class instance methods: NC returns negative values (e.g. -3.1) and AT returns all zeros. Fixed by using `|` (absolute value) for the NC check and conditioning the AT check to only apply when NC is positive.

3. Integration tests use `ThreadMode 'DEBUG'` (overriding the constructor's `ThreadMode 0`) to prevent the server from blocking the dyalogscript process. This matches the pattern used in Stark's own test runner.

4. RegisterTestRoutes and _BrokenHandler are included in TodoApp but have no integration tests in this phase. OnErrorFn testing is deferred to Phase 7 (issue #8) per the implementation plan.
