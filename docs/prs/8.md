## Design decisions

1. Renamed `_BrokenHandler` to `BrokenHandler` (without leading underscore). Stark treats handler names starting with `_` as internal handlers and dispatches them on the Stark instance itself (`(⍎3⊃r)req` at Stark.aplc line 182), not on the Handlers namespace. Since `_BrokenHandler` lived on TodoApp (the Handlers target), Stark could not find it. Without the underscore, Stark dispatches it via `(Handlers⍎3⊃r)req` (line 189), which correctly finds it on TodoApp.

2. Handler audit confirmed: all API handlers (ListTodos, CreateTodo, ToggleTodo, DeleteTodo) use RespondHTML as their final step. Static file handlers (ServePage, ServeCSS) use SetContentType via ContentTypeForFile. All error paths (400, 404) return HTML fragments via RespondHTML. HandleError returns a generic message via RespondHTML.
